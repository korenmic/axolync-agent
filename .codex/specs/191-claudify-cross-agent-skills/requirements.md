# Requirements - Claudify Cross-Agent Skills

Derived from seed `project-seeds/191-p1-claudify-cross-agent-skills.md`.

## R1. Canonical sources stay Codex-shaped

- The tracked skill sources under `skills-workspace/` and `skills-user/` remain the single source of truth.
- Cross-skill invocations in tracked sources use only the Codex `$name` prefix. The Claude `/name` form must never appear as an invocation in tracked sources.

## R2. Claudify generates Claude-form copies

- A single cross-platform Python script `claudify.py` generates Claude-compatible copies of the skills by transforming known-skill `$name` invocations into `/name`.
- The transform rewrites only tokens whose name is an actual known skill (a skill directory name). It must never rewrite any other `$` usage (PowerShell `$env:`/`$var`, bash `$1`, regex, or non-skill `$`), and must never rewrite `/name` inside file paths.
- Copies are hard copies, never symlinks or junctions.

## R3. Output location

- All generated artifacts are written into a gitignored output directory under the agent repo (default `.claudify-out/`), containing `skills-workspace/` and `skills-user/` subtrees.
- The output directory is ignored by git and never tracked.

## R4. Workspace vs user install behavior

- After generating, the script auto-installs (replaces) the workspace-skill copies into the workspace `.claude/skills/` directory on every run.
- User-skill copies are generated into the output directory only. The script must never auto-install them into the global `~/.claude/skills`.
- On every completed run, the script prints an explicit offer to install the user-skill copies globally by hand, including the exact source and destination paths.

## R5. Cross-agent bootstrap documentation

- The README and bootstrap skill docs branch on the active agent: Codex uses `.codex/skills` + `$name`; Claude uses `.claude/skills` + `/name`, generated by claudify.

## R6. CI guarantees

- CI runs in `axolync-agent` GitHub Actions.
- A no-over-reach test proves the transform changes only known-skill invocations: reverse-transforming every generated file reproduces its source byte-for-byte.
- A coverage test proves every skill in both buckets produces a corresponding generated output, so adding a new skill cannot silently break generation.
- Any generation failure, over-reach, or missing output is a CI failure.

## R7. Rogue invocation escaping

- Codex sources never intentionally use the Claude `/name` invocation form, so any pre-existing invocation-position `/name` for a known skill is stale/rogue.
- Claudify neutralizes each such rogue by inserting one space (`/name` -> `/ name`) before converting `$name` -> `/name`, so it cannot trigger a Claude skill.
- Escaping is scoped to invocation position only; file paths (e.g. `skills-user/queue-status`) are never touched.
- A test proves rogue escaping happens and stays reversible; the no-over-reach test proves generated output contains no un-escaped rogue.

## R8. Per-file uninventoried allowlist

- Uninventoried `$`-candidates (documentation placeholders such as `$name`) are allowlisted per file.
- A CI test fails when any uninventoried `$`-candidate appears in a file that does not allow it, so a typo'd or unexpected candidate is surfaced for human review.
