# Enqueue

Use this skill to append relevant undone tasks to the current workspace queue.

This skill may mutate only the local queue file, normally:

`<workspace-root>/.codex/local-task-queue.md`

It must not:

- implement queued tasks
- run TACTIC
- mark tasks complete
- reorder existing queue records
- enqueue unrelated discoverable backlog work

## Scope

Resolve the intended tasks from the current conversation. Examples:

- tasks just generated by `$s2s`
- tasks explicitly named by number or title
- a specific spec or group of specs named by the user
- a backlog task list named by the user
- approved by-value task text that exists only in conversation context

Do not scan every visible spec or backlog for all unchecked tasks. Enqueue only tasks that are clearly in scope from the user request and current context.

## Workflow

1. Resolve the active workspace root.
2. Resolve the intended source tasks from the prompt and current context.
3. Create `<workspace-root>/.codex/local-task-queue.md` if it is missing.
4. Append only undone, non-duplicate tasks using the local queue format.
5. Run `$queue-status verbose`.
6. Add a manual note after queue-status output:

```text
Added N undone tasks in this enqueue session.
Skipped D duplicate tasks.
```

Omit the duplicate line when `D` is zero.

## Queue Record Format

By-reference task:

```markdown
### Q-123
- Status: `queued`
- Source: [tasks.md](/c:/path/to/tasks.md)
- Task: `1. Implement the task`
```

By-value task:

```markdown
### Q-123
- Status: `queued`
- Source: `inline procedural queue task`
- Task: `Implement the approved context-only task with enough detail to execute later.`
```

## Duplicate Rule

Skip a task that is already queued from the same source task identity.

Baseline identity:

- canonical source path
- task number or normalized task title

This source task identity is different from `qid`, which identifies the queue record.

Do not use broad fuzzy matching. Normalize only obvious Markdown wrappers, checklist markers, backticks, and whitespace.
